# Obsidian Salt Dashboard 配置与操作指南

欢迎使用 **Salt Dashboard** 插件。本指南将帮助您快速了解仪表盘的配置方式和各个模块的使用方法。

## 1. 基础操作与布局

### 1.1 开启仪表盘

- **自动打开**: 在插件设置中的「通用设置 (General)」里，勾选 `Open Dashboard on Startup`，即可在每次启动 Obsidian 时自动打开仪表盘。
- **手动打开**: 使用 Obsidian 的命令面板 (Ctrl/Cmd + P)，搜索并执行 `Salt Dashboard: Open Dashboard`，或点击侧边栏对应的图标。

### 1.2 布局交互 (网格系统)

Salt Dashboard 采用响应式 12 列网格布局：

- **拖动模块**: 鼠标按住模块右上角的**拖拽手柄 (Grip 图标)** 进行拖动。为了防止误触选中文字，只有拖动手柄才能移动模块位置。
- **调整大小**: 鼠标悬停在模块的边缘或右下角，拖动以调整模块占据的列数和高度。
- **重置布局**: 如果布局被打乱或想恢复默认样式，可以在设置面板的「布局 (Layout)」部分点击 `Reset to Default Layout` 一键重置。

---

## 2. 全局设置 (General Settings)

在插件设置页的顶部，您可以找到全局设置：

- **TODO Source Folder (待办数据源文件夹)**: 定义所有的待办事项 (Daily/Regular/Jottings) 存放在哪个文件夹（例如 `TODO`）。
- **Global Filter (全局过滤器)**: 用于在“近期文档”和“随机笔记”等模块中排除特定文件夹。采用 Dataview 风格查询，如：`(-"SCRIPTS") and (-"Templates")`，避免抓取到不相干的文件。

---

## 3. 核心模块说明与配置

在「活动模块管理 (Active Modules)」中，可以一键开启或关闭某个模块。关闭后对应的配置项也会自动隐藏。

### 3.1 Date & Progress (日期与进度)

展示当前日期、日历、年度/季度/周进度条，并可统计 Vault 内的笔记数据。

- **配置项**:
  - `Show Stats`: 勾选以显示库内笔记的新增与修改统计（今日/本周/总数）。
  - `Exclude Folders`: 设置不需要进行统计的文件夹名称列表。

### 3.2 Contribution Graph (活跃度图表)

GitHub 风格的年度热力图，展示您的任务完成情况。

- **配置项**:
  - `Color Scheme`: 供选择 `github` 或 `monokai` 的配色。
  - `Click Behavior`: 控制点击热力图方块的动作：
    - `modal`: 弹窗显示当天的所有任务详情。
    - `jump`: 直接跳转到对应的任务源文件。
    - `filter`: 在当前的 Todo 模块中筛选出那一天的任务。
  - `Data Sources`: 您可以勾选此图表统计哪些来源的数据（日常待办/常规待办/随手记）。

### 3.3 Todo 管理系列模块

Salt 提供三种不同形式的 Todo 模块，满足各类任务管理需求：

**① Daily Todo (日常待办)**

- 适合固定重复的日常任务（例如：吃药、打卡、背单词）。
- 带有极简的 crontab 语法编辑器，允许控制任务在每个月的哪几天或星期几显示。
- **配置项**: 指定日常任务的文件名 (如 `daily.md`)，并可决定是否启用 crontab 语法和配置状态缓存文件路径。

**② Regular Todo (常规待办)**

- 适合按天独立的任务规划（例如每天自动创建一个 `260302.md` 用于存放当天的琐碎任务）。
- 模块里可以直接打勾或放弃，自动修改源文件内容。
- **配置项**: 设置日期文件的命名格式（默认 `YYMMDD`），以及是否在没有今日文件时**自动创建文件**。

**③ Jottings Todo (随手记待办)**

- 适合阅读或随手记录时产生的待办事项。当任务内容包含指向特定文件夹（如 `[[jottings/idea]]`）的链接时会自动被此模块抓取。
- **配置项**:
  - 设定 Jottings 的目标文件夹。
  - 设置完成 (`doneTag`) 和放弃 (`abandonedTag`) 时，为对应源文件自动添加的标签（如 `#jottings/done`）。

### 3.4 Recent Files (近期文档)

按列展示最近修改或创建的笔记，支持多列分组显示。

- **配置项**:
  - 可以添加或删除列，每列可配置自定义标题、读取数量和特定的过滤查询语句（例如仅显示 `#diary` 的文件）。
  - 支持配置优先使用 Frontmatter 里的某个特定字段（如 `modified_date`）作为文档的时间依据。

### 3.5 Random Note (随机笔记)

用来在仪表盘上展示一篇随机过去的笔记预览。

- **配置项**:
  - 配置检索源，例如只从 `Notes/` 文件夹中抽取。
  - 自定义卡片显示的高度及桌面/移动端的预览文字长度。

### 3.6 Tmp Note (临时笔记)

一个始终常驻的、纯文本或 Markdown 渲染的便捷暂存区。

- **配置项**: 指定该临时笔记的文件路径（如 `tmp/notes.md`）和显示的固定高度。

## 4. 外部插件扩展 (Extensions)

Salt Dashboard 采用微内核设计，支持从外部加载自定义模块：

- **自定义插件文件夹**: 在设置的 `Custom Plugin Folder` 中填入你的自定义组件所在目录。
- **Reload Extensions**: 编写或修改了外部 JSX/JS 插件后，点击此按钮可热重载并应用新模块。
- **Clean Orphaned Configs**: 若你卸载了某些模块，可点击该按钮清理残留的布局配置文件，保持设置干净。

---

## 5. 危险区域 (Danger Zone)

在设置最底部，提供以下操作：

- **Restore Default Settings (恢复初始设置)**: 一键将整个插件的设置（包括数据、布局和全局选项）重置为最初始的状态。（需点击两次进行二次确认以防误触）。
- **Undo (撤销)**: 如果您刚刚重置了设置，这里会出现一个撤销按钮，允许您恢复到重置前的状态。
